从 Props Hell 到 Compound Pattern：React 公共组件库的重构之路

背景

封装公共组件库的时候时常会遇到一个问题。

当下的设计是合适的。但是随着项目开发的进展，UI 的设计会出现变动。

比如一开始这个组件只需要展示 title 和 description:

type CardProps = {
  title: string;
  description: string;
};
const Card = ({ title, description }: CardProps) => {
  return (
    <div style={{ border: "1px solid black" }}>
      <h2>{title}</h2>
      <p>{description}</p>
    </div>
  );
};

function App() {
  return <Card title="Hello" description="World" />;
}

但是过了两个月后，来了一个新的需求，在某个场景下可以修改这个卡片的内容。 所以我们需要：

  1. 增加一个 Edit Button。
  2. 点击 Edit Button 的时候，弹出弹框对 description 进行修改。

那么为了兼容旧的使用，我们不得不在组件里新增了两个 optional props，isEditable 和 updateDescription。

type CardProps = {
  title: string;
  description: string;
  isEditable?: boolean;
  updateDescription?: (newDescription: string) => void;
};

const Card = ({
  title,
  description,
  isEditable,
  updateDescription,
}: CardProps) => {
  const [isOpen, setIsOpen] = useState(false);
  const [newDescription, setNewDescription] = useState(description);

  const handleDescriptionChange = (e: React.ChangeEvent<HTMLInputElement>) => {
    setNewDescription(e.target.value);
  };
  const handleConfirm = () => {
    updateDescription?.(newDescription);
    setIsOpen(false);
  };

  return (
    <div style={{ border: "1px solid black" }}>
      <h2>{title}</h2>
      <p>{description}</p>
      {isEditable && <button onClick={() => setIsOpen(true)}>Edit</button>}
      {isOpen && (
        <dialog open>
          <input
            type="text"
            value={newDescription}
            onChange={handleDescriptionChange}
          />
          <button onClick={handleConfirm}>Confirm</button>
        </dialog>
      )}
    </div>
  );
};

function App() {
  const [data, setData] = useState({
    title: "New Hello",
    description: "New World",
  });
  const updateDescription = (newDescription: string) => {
    setData({ ...data, description: newDescription });
  };
  return (
    <>
      {/* 旧的使用 */}
      <Card title="Hello" description="World" />
      {/* 新的使用 */}
      <Card
        title={data.title}
        description={data.description}
        isEditable={true}
        updateDescription={updateDescription}
      />
    </>
  );
}

可以想象, 再过两个月，又来一个新需求，要求弹出的提示框的内容根据场景指定，那么我们的 Card 组件又要新增一个叫做 dialogDescription 的 optional prop 了。
